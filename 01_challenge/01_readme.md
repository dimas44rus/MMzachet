# Challenge 01 - Отчет об ошибке

### Что делает функция в этом задании?
Функция реализует классический алгоритм FizzBuzz: перебирает целые числа в диапазоне, выводит слово `fizz`, если число кратно 3, слово `buzz`, если оно кратно 5, и совмещенную строку, если число делится на оба значения одновременно.

### 1. Что подсказало решение в тексте ошибки?
Интерпретатор Python выдал `SyntaxError: invalid syntax` и указал стрелочкой на строку 27 в конструкции `elif i%num1=0:`.

### 2. Как была решена проблема?
Внутри оператора ветвления `elif` использовался одиночный знак равенства `=`, который служит оператором присваивания. Для проверки равенства (сравнения чисел) необходимо использовать двойное равенство `==`. Ошибка исправлена путем замены `=` на `==`.

### 3. Чему я научился?
Я вспомнил базовое отличие синтаксиса Python: `=` — это присваивание значения переменной, а `==` — это сравнение двух выражений.
