# Отчет по зачетной работе: Поддержка и тестирование программных модулей

Репозиторий содержит выполнение практического задания по отладке и тестированию кода на языке Python. В рамках работы была изучена серия из 13 челленджей (проект `wtfiswronghere`), моделирующих реальные ошибки программирования от простых синтаксических опечаток до комплексных логических багов.

## 🎯 Цели и задачи работы
- Развить навыки чтения трассировки ошибок (Traceback).
- Освоить локализацию дефектов в коде и работу со стеком вызовов.
- Закрепить базовые концепции Python (типы данных, отступы, ООП, импорт модулей).
- Получить практический опыт работы с системой контроля версий Git и платформой GitHub.

## 🛠️ Сводная таблица исправленных ошибок

| Папка | Тип ошибки | Краткое описание проблемы и решение |
| :--- | :--- | :--- |
| **01** | `SyntaxError` | Одиночное равно `=` вместо `==` в условии `elif`. Исправлено на оператор сравнения. |
| **02** | `TypeError` | Вызов функции `fizzbuzz()` без обязательного аргумента `max_num`. Добавлено значение `100`. |
| **03** | `TypeError` | Передача числа в виде строки `"100"` в кавычках. Кавычки удалены для приведения к типу `int`. |
| **04** | `IndentationError` / `TabError` | Нарушение отступов в блоке условий и смешивание табов с пробелами. Выровнено пробелами. |
| **05** | `SyntaxError` / `FileNotFoundError` | Синтаксис `print` из Python 2 и опечатка в имени файла (`mifile` вместо `myfile`). Исправлено. |
| **06** | `AttributeError` | Опечатка при обращении к атрибуту модуля (`conf.num` вместо `conf.num2`). Исправлено. |
| **07** | Логический баг | Цикл останавливался на 98 из-за вызова `fizzbuzz(99)`. Аргумент заменен на `100`. |
| **08** | `TypeError` | Пропущен обязательный аргумент `self` в объявлении метода класса. Добавлен `self`. |
| **09** | Логический баг | Начало диапазона цикла было установлено с 6 (`range(6, ...)`). Исправлено на `1`. |
| **10** | `TypeError` | Попытка вызвать импортированный модуль как функцию. Импорт заменен на `from fizzbuzz import fizzbuzz`. |
| **11** | Логический баг | Неверный порядок условий `if/elif`. Комплексная проверка кратных чисел вынесена на первое место. |
| **12** | Логический баг | В конструкторе `__init__` переменная `self.num2` была равна 4 вместо 5. Значение исправлено. |
| **13** | Комбо (`SyntaxError` / `NameError`) | Незакрытая кавычка, пропущенное `:` в цикле `for` и опечатка в имени функции (`fizzbuzzy`). Всё устранено. |

## 🎓 Чему я научился
1. **Эффективному дебаггингу:** Быстро определять характер проблемы по тексту исключения в терминале.
2. **Внимательности к деталям:** Пониманию того, что программа может работать без системных ошибок, но при этом выдавать неверный бизнес-результат (как в челленджах 7, 9, 11 и 12).
3. **Культуре ведения Git:** Каждое исправление было зафиксировано отдельным атомарным коммитом с понятным описанием проделанной работы.
